01 Application Insights pour .NET 10

  • Atelier pratique bilingue (FR / EN)
  • Navigateur → Web ASP.NET Core → API Web → Azure SQL
  • Un seul operation_Id, une seule cascade, une seule vérité

Accueillez la salle dans les deux langues. Confirmez les règles bilingues avant de commencer.

02 Ordre du jour — six modules

  • Module 0 — Préparation et prérequis (15 min)
  • Module 1 — Provisionnement (azd up) (30 min)
  • Module 2 — Auto-instrumentation Web (30 min)
  • Module 3 — Instrumentation API + SQL (45 min)
  • Module 4 — Corrélation côté navigateur (30 min)
  • Module 5 — Tableaux de bord et KQL (30 min)
  • Module 6 — Démantèlement (azd down --purge --force) (10 min)

~3 h au total avec pauses. Modules à respecter strictement; le cœur est Module 4 + Module 5.

03 Objectifs d'apprentissage

  • Provisionner Application Insights avec Bicep + azd en une seule commande
  • Auto-instrumenter une application Web ASP.NET Core 10 sans une ligne de code
  • Corréler pageView navigateur → requête serveur → dépendance SQL sur un même operation_Id
  • Écrire une requête KQL qui joint requests, dependencies et pageViews de bout en bout
  • Démanteler proprement toutes les ressources avec azd down --purge --force

Ancrez les attentes. Chaque objectif est renforcé par un atelier pratique.

04 Module 0 — Prérequis

  • .NET SDK 10.0.100 (préversion ou RTM)
  • Azure Developer CLI (azd) 1.10+
  • Azure CLI (az) 2.65+
  • VS Code avec les extensions C# Dev Kit et Bicep
  • Abonnement Azure actif avec rôle Owner ou Contributor + UAA sur un groupe de ressources
  • Compte GitHub (pour le carnet ensemencé et les flux CI)

Si quelqu'un n'a pas le SDK, dirigez-le vers le devcontainer (lab-00).

06 Module 1 — Provisionnement (azd up)

  • azd init -t . (utilisez le modèle de l'atelier)
  • azd env new mapaq-dev
  • azd up (Bicep déploie RG, AI, Log Analytics, App Service, SQL)
  • Vérifiez les sorties : APPLICATIONINSIGHTS_CONNECTION_STRING est câblé sur les deux apps

Comptez ~6 min pour le premier déploiement. La chaîne de connexion est un App Setting, jamais en dur dans le code.

07 Modules Bicep de l'atelier

  • infra/main.bicep — compose les modules
  • infra/modules/log-analytics.bicep — fondation workspace
  • infra/modules/app-insights.bicep — composant basé sur le workspace
  • infra/modules/app-service.bicep — Web + Api avec identité managée système
  • infra/modules/sql.bicep — Azure SQL avec authentification Entra uniquement
  • Passez la chaîne de connexion via un App Setting, jamais en dur

Insistez : AI est désormais basé sur un workspace par défaut; AI classique est déprécié.

08 Module 2 — Auto-instrumentation Web

  • dotnet add package Microsoft.ApplicationInsights.AspNetCore
  • builder.Services.AddApplicationInsightsTelemetry();
  • Lit APPLICATIONINSIGHTS_CONNECTION_STRING depuis la configuration automatiquement
  • Génère requests, dependencies, exceptions et traces sans code supplémentaire

C'est volontairement ennuyeux. La magie arrive au prochain module — la propagation.

09 Ce que vous devriez voir en 60 secondes

  • Live Metrics affiche les requêtes entrantes en temps réel
  • Transaction Search liste chaque requête avec son operation_Id
  • La vue transaction de bout en bout regroupe les dépendances sous leur requête parente
  • Le tableau Failures s'allume dès qu'un 5xx remonte de l'API

Gardez Live Metrics ouvert pendant la démo; rien ne convainc plus vite que les points qui apparaissent.

10 Module 3 — Instrumentation API + SQL

  • Même paquet Microsoft.ApplicationInsights.AspNetCore sur l'API
  • Les appels SQL EF Core 10 sont suivis automatiquement comme dépendances SqlClient
  • HttpClientFactory propage le traceparent vers les appels en aval automatiquement
  • Dimensions personnalisées : TelemetryClient.TrackEvent("OrderPlaced", new { OrderId, CustomerId })

L'instrumentation EF Core ⇒ SqlClient ne demande aucun paquet supplémentaire — elle est livrée avec le SDK AI.

11 W3C Trace Context — traceparent

  • 00-{trace-id}-{parent-id}-{trace-flags}
  • Le SDK navigateur démarre la trace et émet l'en-tête sur chaque fetch/XHR
  • ASP.NET Core l'honore et le transmet sans configuration
  • Résultat : un seul operation_Id relie les 4 spans ensemble

Montrez l'en-tête réel dans l'onglet Réseau des DevTools du navigateur pendant la démo.

13 Module 4 — Corrélation côté navigateur

  • Ajoutez le snippet du SDK JavaScript Application Insights à _Layout.cshtml
  • Passez la chaîne de connexion via un ViewData côté serveur (jamais en dur)
  • Activez enableCorsCorrelation: true et enableRequestHeaderTracking: true
  • Passez distributedTracingMode: AI_AND_W3C pour émettre traceparent

Le snippet doit venir avant tout autre script qui émet fetch/XHR.

14 SDK navigateur — configuration minimale

  • instrumentationKey est obsolète — utilisez connectionString
  • enableAutoRouteTracking: true (pour les changements de page SPA)
  • correlationHeaderExcludedDomains: ['*.googleapis.com', '*.bing.com']
  • samplingPercentage: 100 en dev, plus bas en prod

L'échantillonnage est par app, fixé sur l'objet d'options du SDK — pas dans le portail.

16 KQL — une seule requête, un seul operation_Id

  • Ouvrez le panneau Logs sur la ressource AI
  • Exécutez la requête union ci-dessous
  • Triez par timestamp asc pour voir la cascade dans l'ordre temporel
flowchart LR
  Q["union requests, dependencies, pageViews
| where operation_Id == 'abc'
| project timestamp, itemType, name, duration, success
| order by timestamp asc"]
  Q --> R["📊 Résultat cascade
4 lignes · 1 operation_Id"]

Décortiquez chaque ligne. L'union sur 3 tables est l'astuce que la plupart des nouveaux ratent.

17 Requête KQL — copier / coller

  • union requests, dependencies, pageViews
  • | where operation_Id == 'abc-trace-id-ici'
  • | project timestamp, itemType, name, duration, success, target
  • | order by timestamp asc

Remplacez 'abc-trace-id-ici' par l'operation_Id récupéré dans l'en-tête traceparent des DevTools.

18 Live Metrics — l'effet whaouh

  • Latence inférieure à la seconde entre l'app et le portail
  • Aucune donnée n'est persistée dans le flux du portail — l'échantillonnage ne s'applique pas
  • Filtrez par nom de rôle (cloud_RoleName) pour isoler Mapaq.Api
  • Génial pour la démo; ne remplace pas de vraies alertes

Ouvrez Live Metrics, puis demandez à la salle de charger la page — dopamine garantie.

19 Workbook — synthèse SLO bilingue

  • Workbook prêt à l'emploi dans workshop-content/workbooks/
  • Affiche le volume de requêtes 24h, p95 de latence, % d'échec, top-5 dépendances
  • Paramétré par nom de rôle et plage de temps
  • Importable via JSON, sans Power BI

Le JSON du workbook vit dans le dépôt; importez via la galerie de la ressource AI.

20 Alertes intelligentes — anomalies de panne

  • Smart Detection s'exécute en continu sur le workspace
  • Détecte automatiquement les anomalies de taux d'échec, de chargement de page lent et de dégradation de trace
  • Configurez l'e-mail + un Action Group pour un webhook Teams

Smart Detection est actif par défaut — montrez comment ajouter des destinataires.

21 Maîtrise des coûts — sans surprise

  • Plafond quotidien sur la ressource AI (50 Mo/jour pour l'atelier)
  • Échantillonnage : 100 % en dev, 10–25 % typique en prod
  • AI basé sur le workspace utilise les paliers d'engagement Log Analytics
  • Étiquetez tout avec Workshop=Mapaq pour la facturation

Le plafond quotidien protège l'abonnement de l'atelier d'un débordement de logs.

22 Liste de dépannage

  • Pas de télémétrie ? Vérifiez APPLICATIONINSIGHTS_CONNECTION_STRING dans les App Settings
  • PageViews manquants ? Confirmez que le snippet est avant les autres scripts
  • operation_Id différent ? Vérifiez enableCorsCorrelation sur le SDK JS
  • Dépendances SQL absentes ? EF Core a besoin du SDK AI — le paquet seul ne suffit pas

Gardez cette diapo visible pendant le lab; la plupart des questions tombent dans une de ces 4 lignes.

23 Module 6 — Démantèlement

  • azd down --purge --force
  • --purge supprime les Key Vaults et ressources AI en suppression douce
  • --force saute la confirmation interactive
  • Vérifiez dans le portail Azure que le groupe de ressources est bien parti

Démontrez toujours le démantèlement en direct. Un atelier sans démantèlement est une bombe à retardement de facturation.
